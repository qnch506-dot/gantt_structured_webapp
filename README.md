# Gantt Structured Web App

Excel 백데이터를 업로드하면 다음 흐름으로 간트차트 Excel을 생성하는 샘플 프로젝트입니다.

1. Excel 업로드
2. 백데이터 컬럼 분석
3. 시작일/종료일 검증
4. 날짜가 있으면 그대로 반영
5. 날짜가 없고 작업일이 있으면 자동 배치
6. 간트차트 Excel 생성
7. 결과 파일 다운로드

## 백데이터 권장 형식

| 이름 | 작업명 | 작업일 | 시작일 | 종료일 |
|---|---|---:|---|---|
| 가설공사 | 가설울타리 설치 | 10 | 2026-01-01 | 2026-01-10 |
| 토공사 | 터파기 | 20 | 2026-01-08 | 2026-01-27 |

선택 컬럼:

| 컬럼 | 설명 |
|---|---|
| 진행률 | 0~100 또는 0~1 |
| 선행작업 | 앞 작업명 |
| 층 | 층별 공정 표시용 |
| 구역 | 구역별 공정 표시용 |

## 실행 방법

### 1. Backend 실행

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 2. Frontend 실행

새 CMD 창에서:

```bash
cd frontend
npm install
npm run dev
```

브라우저:

```text
http://localhost:3000
```

## API

### POST `/api/preview`

Excel 파일을 분석하고, 작업별 상태를 반환합니다.

### POST `/api/generate`

Excel 파일을 간트차트 Excel로 변환해 다운로드합니다.

### POST `/api/upload`

기존 업로드 흐름 호환용입니다. 업로드 즉시 간트차트 Excel을 반환합니다.

## 핵심 설계

- 시작일/종료일이 있는 작업: 날짜 그대로 사용
- 시작일만 있고 작업일이 있는 작업: 종료일 자동 계산
- 종료일만 있고 작업일이 있는 작업: 시작일 자동 계산
- 시작일/종료일이 없고 작업일만 있는 작업: 자동 배치
- 작업일도 없으면 오류 처리


## v3 수정 사항

이번 버전에는 다음 기능을 추가했습니다.

1. 웹페이지 우측에 Excel 스타일 간트차트 미리보기 UI 추가
2. 미리보기에서 일별 / 주간별 / 월간별 탭 전환 가능
3. 다운로드되는 Excel 결과 파일에 `일별`, `주간별`, `월간별` 시트 자동 생성
4. 작업일 또는 기간이 `0`인 작업은 오류가 아니라 마일스톤으로 처리

시트 기준:

| 시트 | 기준 |
|---|---|
| 일별 | 하루 = 한 칸 |
| 주간별 | 1주 = 한 칸 |
| 월간별 | 1개월 = 한 칸 |

주간별과 월간별은 일별 간트차트를 압축해서 보는 용도입니다.


## v4 수정 사항

간트차트 바 스타일을 셀 전체 색상 채움 방식에서 화살표 선 방식으로 변경했습니다.

표시 방식:

| 구분 | 표시 |
|---|---|
| 시작 | ◀ |
| 중간 기간 | ━ |
| 종료 | ▶ |
| 시작과 종료가 같은 칸 | ↔ |
| 작업일 0 마일스톤 | ◆ |

이 방식은 셀 배경을 꽉 채우지 않아서, 원본 공정표처럼 그리드 안쪽에 얇은 화살표가 있는 느낌으로 표시됩니다.


## v5 수정 사항

화살표 색상 기준을 `작업명`이 아니라 `이름/공종` 기준으로 변경했습니다.

예시:

| 이름 | 작업명 | 화살표 색상 |
|---|---|---|
| 가설공사 | 가설울타리 | 같은 색 |
| 가설공사 | 가설사무소 | 같은 색 |
| 토공사 | 터파기 | 다른 색 |
| 토공사 | 잔토처리 | 같은 색 |

적용 범위:

1. 웹페이지 간트차트 미리보기
2. 다운로드 Excel의 일별 시트
3. 다운로드 Excel의 주간별 시트
4. 다운로드 Excel의 월간별 시트


## v6 수정 사항

화살표 중간선이 얇고 끊어져 보이는 문제를 개선했습니다.

수정 내용:

1. 웹 미리보기는 문자 `━` 대신 실제 CSS 선으로 표시
2. 선 두께를 3px로 키움
3. 셀 경계에서 끊겨 보이지 않도록 선을 좌우로 살짝 확장
4. Excel 결과는 중간선을 `━━━━` 형태로 굵게 표시
5. Excel 화살표 폰트 크기와 칸 폭을 약간 키움

화살표 끝 모양은 유지하고, 중간 기간선만 더 굵고 이어진 느낌으로 조정했습니다.


## v7 수정 사항

좌측 `이름 / 작업명` 표시 구조를 공정표 형식에 가깝게 변경했습니다.

수정 내용:

1. Excel 결과 파일에서 같은 `이름/공종`이 연속될 경우 A열을 세로 병합
2. 작업명은 같은 이름/공종 내부에서 `1. 작업명`, `2. 작업명` 형식으로 표시
3. 이미 작업명에 `1.`, `2.` 같은 번호가 있으면 중복 번호를 붙이지 않음
4. 웹페이지 간트차트 미리보기에서도 같은 이름/공종은 세로 병합 느낌으로 표시
5. 이름 컬럼은 좁게, 작업명 컬럼은 더 넓게 조정

적용 범위:

- 웹페이지 간트차트 미리보기
- Excel 일별 시트
- Excel 주간별 시트
- Excel 월간별 시트


## v8 수정 사항

Excel 결과 파일의 화살표 표시 방식을 완전히 변경했습니다.

기존 문제:

1. 일별 시트에서 셀마다 문자 화살표가 들어가서 선이 끊겨 보임
2. 주간별/월간별도 실제로는 한 줄 화살표가 아니라 칸별 문자 표시였음

수정 내용:

1. 일별/주간별/월간별 시트 모두 투명 PNG 화살표를 셀 위에 올리는 방식으로 변경
2. 하나의 작업 기간이 하나의 연속된 화살표처럼 표시됨
3. 셀 배경을 채우지 않아서 기존 그리드가 보임
4. 같은 이름/공종은 같은 색 화살표 유지
5. 기간 0은 ◆ 마일스톤 PNG로 표시

주의:

- 이 버전부터 Excel 화살표 생성을 위해 `Pillow` 패키지가 필요합니다.
- `pip install -r requirements.txt`를 다시 실행해야 합니다.


## v9 수정 사항

대용량 백데이터 파일 테스트를 반영했습니다.

테스트 파일: `실제형_건축간트_백데이터_대용량_582개(1).xlsx`

수정 내용:

1. `설명_요약`, `공종별_작업수` 같은 설명/집계 시트는 작업 데이터로 읽지 않도록 제외
2. 실제 작업 데이터 582개만 정상 인식
3. 기존처럼 설명 행이 오류로 잡히는 문제 수정
4. Excel 생성 속도 최적화
   - 모든 날짜 셀에 border/fill을 칠하지 않고 Excel gridline 사용
   - 작업 구간은 연속 PNG 화살표로 표시
5. 웹 미리보기는 속도 보호를 위해 상위 150개 작업만 표시
6. 다운로드 Excel에는 전체 작업이 모두 포함됨

검증 결과:

| 항목 | 결과 |
|---|---|
| 인식 작업 수 | 582개 |
| 오류 작업 수 | 0개 |
| 생성 시트 | 일별 / 주간별 / 월간별 |
| 대용량 생성 테스트 | 성공 |


## v10 수정 사항

일별 / 주간별 시트에서 화살표가 셀 경계에 걸쳐 보이는 문제를 수정했습니다.

수정 내용:

1. 일별 화살표 이미지에 좌우 내부 여백 추가
2. 주간별 화살표 이미지에 좌우 내부 여백 추가
3. 이미지 앵커를 셀 좌상단 고정 + 오프셋 방식으로 변경
4. 월별 시트는 기존 방식 유지

결과적으로 일별/주간별 화살표가 칸 바깥으로 걸쳐 보이지 않고,
셀 안쪽에 더 안정적으로 배치됩니다.


## v11 수정 사항

Excel 결과 파일에서 날짜/주간/월간 헤더에 생기던 필터 드롭다운 화살표(▼)를 제거했습니다.

수정 내용:

1. 필터 적용 범위를 전체 간트차트 영역에서 `A:E` 좌측 정보 컬럼으로 축소
2. `이름 / 작업명 / 작업일 / 시작일 / 종료일`에는 필터 유지
3. 일별 날짜 칸에는 필터 화살표 제거
4. 주간별 주차 칸에는 필터 화살표 제거
5. 월간별 월 칸에는 필터 화살표 제거


## v12 수정 사항

다운로드 시 발생하던 `OneCellAnchor.ext` 오류를 수정했습니다.

오류 원인:

`openpyxl`의 `OneCellAnchor.ext`에는 `(width, height)` 튜플이 아니라
`XDRPositiveSize2D` 객체를 넣어야 합니다.

수정 내용:

1. `XDRPositiveSize2D` import 추가
2. 이미지 anchor 크기 지정 방식을 tuple에서 `XDRPositiveSize2D(cx, cy)`로 변경
3. 일별/주간별/월간별 PNG 화살표 이미지 다운로드 오류 수정


## v13 수정 사항

일별 / 주간별 시트에서 화살표 머리가 칸 경계에 걸치던 문제를 수정했습니다.

수정 내용:

1. 일별 화살표 시작점 = 첫 날짜 칸의 중앙
2. 일별 화살표 종료점 = 마지막 날짜 칸의 중앙
3. 주간별 화살표 시작점 = 첫 주차 칸의 중앙
4. 주간별 화살표 종료점 = 마지막 주차 칸의 중앙
5. 마일스톤도 해당 칸 중앙에 오도록 보정
6. 월별은 기존 표시 방식 유지

즉, 이제 화살표 머리가 칸 사이 경계선에 걸치지 않고
각 칸 안쪽 중앙에 더 깔끔하게 들어가도록 조정했습니다.


## v14 수정 사항

일별 / 주간별 화살표의 위치 기준을 다시 수정했습니다.

기존 문제:
- 화살표 선은 대체로 맞았지만
- 화살표 머리 자체가 시작/끝 칸의 정중앙에 오지 않고
- 칸 경계 쪽으로 치우쳐 보였습니다.

수정 내용:
1. 일별 화살표 시작 머리 끝점 = 시작 날짜 칸 정중앙
2. 일별 화살표 종료 머리 끝점 = 종료 날짜 칸 정중앙
3. 주간별 화살표 시작 머리 끝점 = 시작 주차 칸 정중앙
4. 주간별 화살표 종료 머리 끝점 = 종료 주차 칸 정중앙
5. 마일스톤도 해당 칸 정중앙으로 배치
6. 월별은 기존 방식 유지

즉, 선 전체가 아니라 '화살표 머리 자체'가 자기 칸 중앙에 오도록 변경했습니다.


## v15 수정 사항

Excel 다운로드 결과에서 화살표가 작업 행의 위쪽에 붙어 보이던 문제를 수정했습니다.

수정 내용:

1. 화살표 PNG 이미지 높이를 행 전체보다 작게 조정
2. 이미지 row offset을 행 높이 기준 중앙값으로 계산
3. 일별 / 주간별 / 월간별 모두 세로 중앙 배치 보정
4. 작업 행 높이를 26pt로 약간 확장
5. 가로 위치는 기존 v14 기준 유지
   - 일별/주간별: 화살표 머리가 각 시작/끝 칸 중앙
   - 월별: 기존 방식 유지

즉, 이제 화살표가 위쪽 셀 경계에 붙지 않고 자기 작업 행의 세로 중앙에 오도록 보정했습니다.


## v16 수정 사항

요청대로 화살표 좌우 위치는 초기 방식으로 되돌리고, 상하 위치만 수정했습니다.

수정 내용:

1. 좌우 위치 복원
   - 일별: 좌우 3px 여백
   - 주간별: 좌우 4px 여백
   - 월별: 기존 그대로
2. 상하 위치만 보정
   - 화살표 이미지 높이를 더 줄임
   - row offset을 아래로 더 내려 자기 행 중앙에 오도록 조정
3. 작업 행 높이를 28pt로 약간 증가

즉,
- 화살표 좌우 위치는 예전처럼 유지
- 화살표가 위로 붙어 보이던 문제만 개선
